SLICE (PPLNS+JD) Live Slice Builder

SLICE is DMND's payment system — a modified PPLNS designed for Stratum V2. It splits block rewards into two parts: the subsidy (paid by hashrate alone) and the fees (paid by hashrate × the financial value of the blocks you build). This simulator shows that process happening in real time. Read the full explainer →

How this simulator works
01 · Mempool
Fee rate fluctuates over time
The simulator models a live mempool where the fee rate (sat/vB) drifts, spikes, and reverts. Each miner builds blocks using their node — but with different fee efficiency: how close to the maximum available fee rate they capture.
02 · Slicing
Slices close when fees rise enough
A slice is a time window of shares. It closes when the mempool fee rate rises by the threshold % above where the slice started — or when a block is found. All shares inside a slice are scored fairly against the same MMEV benchmark.
03 · Scoring
Score = (your fee rate / MMEV) × difficulty
Each share is weighted by how much fee value it carried, relative to the maximum possible in that slice (MMEV). A miner with 100% efficiency always scores full weight. Lower efficiency means lower score — and therefore a smaller cut of the fee portion.

Simulation Controls

Use speed to run through time quickly — 5× fills the lookback window in about 30 seconds. The fee Δ threshold controls how sensitive the slice trigger is: lower = more slices, finer scoring granularity; higher = fewer, larger slices with more within-slice fee variance.
